# Accessibility Constraints

- Use semantic HTML before ARIA; every page must have one meaningful main landmark.
- Maintain a logical heading hierarchy, visible keyboard focus, and usable color contrast in both configured color schemes.
- Every interactive control needs an accessible name; every form control needs a label and linked error message.
- Use meaningful image alt text; decorative images use empty alt text intentionally.
- Do not rely on hover, color, or pointer precision as the only interaction mechanism.
